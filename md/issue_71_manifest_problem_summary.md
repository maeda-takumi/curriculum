# 71ページ表示不具合の現状整理

## 背景
- `files_71` に、これまでのページで見られた `[docId]-xxxx.js` が存在しない。
- ただしこれは必ずしも異常ではなく、71系は旧形式と異なるビルド構成（ハッシュchunk中心）になっている可能性が高い。

## 観測された事実
- コンソール上の目立つエラーは `POST /cdn-cgi/rum 404`。
  - これはCloudflareのRUM計測送信失敗であり、表示崩れの主因とは限らない。
- `https://schoolai.biz/curriculum/phase/71_files/_buildManifest.js` は取得できる。
- ただし共有された内容では、以下の不自然点がある。
  1. 内容が重複して連結されているように見える（`...CB()self.__BUILD_MANIFEST...`）。
  2. `\n` が文字列として混入している可能性がある。
  3. マニフェスト内の参照が `static/chunks/*.js` 形式で、実配置ファイル群（`turbopack-*.js` など）と整合しない疑いがある。

## 現時点の有力原因
1. **_buildManifest.js の破損（連結/エスケープ混入）**
   - JSとして正しく評価されず、hydration（再描画）が開始できない可能性。
2. **manifest と実ファイルセットの不一致**
   - manifestが要求するchunkが存在しない、または場所が異なる可能性。
3. （補足）`/cdn-cgi/rum 404` はノイズであり、主因ではない可能性が高い。

## 仮説（優先順）
- 第一候補: `_buildManifest.js` が壊れている。
- 第二候補: 71ページ資産の取得/配置が「同一ビルド一式」になっておらず、参照先が噛み合っていない。

## 次アクション（短時間で確認可能）
1. `_buildManifest.js` の実体を確認
   - 重複連結がないか
   - JS構文として成立しているか
   - `\n` が生文字として混入していないか
2. manifest参照先の存在確認
   - 例: `static/chunks/7a3c3fdf2c1f5beb.js` などが同一配下に存在するか
3. 71ページを「単体ファイル」ではなく「同一ビルドの必要資産一式」で再配置
   - HTML + CSS + manifest + runtime + chunk群をセットで揃える

## 結論
- 現状の表示不具合は、`[docId]` ファイル欠如そのものより、
  **build manifest周りの破損または資産不整合により hydration が止まっている** 可能性が高い。
