第1章：WordPressを使う理由と使い分けの判断基準

1-1. WordPress対応が単価とリピート率を同時に上げる

上のグラフを見てください。静的HTML（htmlファイルをサーバーにアップするだけ）の納品では平均単価が15万円程度ですが、WordPress組み込みになると30万円、さらに運用サポートを加えると45万円、定期更新代行まで含めると80万円まで単価が上昇します。
そしてリピート率がより重要です。
静的HTMLのみの納品では20%程度なのに対して、WordPressで運用サポートを組み込むと85%まで跳ね上がります。
なぜここまでリピート率が違うのかというと、静的HTMLを納品した場合はクライアントが「あとは自分で管理します」となり、次の依頼はサイトのリニューアル時（数年後）になりがちです。
一方WordPressで納品し月次の保守・更新サービスを提案した場合、継続的な関係が生まれます。「毎月ブログを2本書いてほしい」「新サービスのページを追加してほしい」という依頼が定期的に来るようになります。
WordPress対応を習得することは、単に案件の幅が広がるだけでなく、副業の安定した月次収入モデルを作ることにも直結します。
1-2. WordPress vs 静的HTML：どちらを選ぶかの判断マトリクス
上のマトリクスを見ると、WordPressと静的HTMLの使い分けが明確になります。
「クライアントが自分でコンテンツを更新したい」「ブログや事例を継続的に追加したい」「SEOを本格強化したい」という条件が当てはまる場合はWordPressが適しています。
反対に「LPのみの1〜2ページ」「納品後の更新がない」「スピード最優先」という場合は静的HTMLで十分です。
実際の案件では最初のヒアリング時に「今後ご自身でブログや事例の更新をされたいですか？」という質問を一つ入れるだけで、ほとんどの場合どちらが適しているかが判断できます。
「できれば自分で更新できるようにしたい」と言えばWordPress一択、「難しいことはすべてお任せします」という場合は静的HTMLで始めて後からWordPressに移行するというアプローチも有効です。
1-3. WordPressの基本的な仕組みを理解する
WordPressとは「サーバー上で動く、Webサイトのコンテンツ管理システム（CMS）」です。通常のHTMLファイルは「完成品をサーバーに置くだけ」ですが、WordPressは「サーバー上にプログラムが動いていて、管理画面からコンテンツを追加・変更するとWebページが自動的に生成される」仕組みです。
この仕組みのおかげで、HTMLの知識がないクライアントでも「ブログ記事を書く感覚でサイトのコンテンツを更新できる」ようになります。
WordPressで最初に理解すべき重要な概念が「テーマ」と「プラグイン」と「固定ページ」の3つです。テーマはサイト全体のデザインと構造を定義するテンプレートです。プラグインはWordPressの機能を拡張するアドオンで、お問い合わせフォームやSEO設定などを追加できます。
固定ページはLP・HP・会社概要ページなど、制作者が作成する静的なコンテンツです。ClaudeのHTMLをこの「固定ページ」に組み込むことがこの章の核心です。

第3章：ClaudeのHTMLをWordPressに組み込む──完全操作手順
3-1. 「body部分だけを貼り付ける」という大原則
上の操作フロー図がWordPressへのHTML組み込みの5ステップです。
このフローを実行する上で最も重要な原則を最初に理解してください。
それは「ClaudeのHTMLのうちWordPressに貼り付けるのはbody部分の中身だけ」ということです。
Claudeが生成するHTMLは完全な構造を持っています。
先頭の<!DOCTYPE html>から始まり、<head>タグ（CSSやJSのリンク・メタ情報）、<body>タグ（実際のページコンテンツ）、末尾の</html>という構造です。
しかしWordPressはすでに独自の<head>と<body>構造を持っているため、Claudeが生成する<head>の内容はWordPressの設定（後述のfunctions.php）で別途読み込みます。
固定ページに貼り付けるのは「<body>タグの直後から</body>タグの直前まで」の部分だけです。この原則を守らないとHTMLが二重になってページが崩れます。
3-2. トップページを固定ページに組み込む（詳細手順）
実際の操作手順を、トップページ（index.html）をWordPressのフロントページとして設定する例で解説します。
これがHP制作の4ページ中で最も設定が複雑です。他のページはこの手順の繰り返しです。

トップページをWordPressに組み込む完全手順
  • 【STEP 1】管理画面にログインして「固定ページ」→「固定ページを追加」をクリック
  • → 画面上部のタイトル欄に「トップページ」と入力
  • （このタイトルはURL・サイト表示には影響しない）
  • 【STEP 2】エディターを「コードエディター」に切り替える
  • → 画面右上の「…」（縦三点ドット）アイコンをクリック
  • → ドロップダウンメニューの中から「コードエディター」を選択
  • → テキスト入力エリアが１つの大きなテキストボックスに変わればOK
  • ※ここで「ビジュアルエディター」のままだとHTMLが正しく機能しない
  • 【STEP 3】index.htmlのbody部分だけをコピーする
  • → VSCodeまたはメモ帳でindex.htmlを開く
  • → <body>タグの行の次から</body>タグの前の行まで選択
  • → Ctrl+C（Mac: Command+C）でコピー
  • 【STEP 4】WordPressのコードエディターに貼り付ける
  • → コードエディターのテキストボックスをクリックしてカーソルを置く
  • → Ctrl+V（Mac: Command+V）で貼り付け
  • 【STEP 5】スラッグ（URL）を設定する
  • →下書き保存する
  • → 画面右側のパネルを見る
  • → 「固定ページ」エリアにある「スラッグ」を確認
  • → トップページはスラッグ不要（フロントページ設定で制御するため）
  • 【STEP 6】「右上💻アイコン」→「新しいタブでプレビュー」で確認する
  • → まずはプレビューで表示を確認してからPublish（公開）する
  • → この時点ではCSS・フォントが未適用のため崩れることがある
  • → 第4章でCSS・フォントを設定すると正しく表示される
  • 【STEP 7】「公開」ボタンをクリックして固定ページを公開する
  • → 確認が取れたら右上の「公開」ボタンをクリック
3-3. フロントページ（トップページ）の設定──必ず行う重要設定
固定ページを作成しただけでは、そのページがサイトのトップ（ルートURL）に表示されるわけではありません。
「このページをサイトのトップとして使います」という設定を管理画面で行う必要があります。初めてWordPressを触る方がよくつまずくポイントです。

フロントページの設定手順（必須）
  • 【STEP 1】左メニューの「設定」→「表示設定」をクリック
  • 【STEP 2】「ホームページの表示」のラジオボタンを確認する
  • → デフォルトは「最新の投稿」になっている
  • → 「固定ページ」のラジオボタンをクリックして変更する
  • 【STEP 3】「ホームページ」のドロップダウンを設定する
  • → 「ホームページ」のドロップダウンをクリック
  • → 先ほど作成した「トップページ」を選択する
  • ※「投稿ページ」はブログがある場合のみ設定。なければ空欄のまま
  • 【STEP 4】「変更を保存」ボタンをクリックする
  • 【STEP 5】ブラウザでサイトのルートURLを開いて確認する
  • → https://クライアントのドメイン/ を開いてトップページが表示されることを確認
  • → 表示されれば設定完了
3-4. パーマリンクの設定
──URLを正しく機能させる
サービスページ・会社概要・お問い合わせの各固定ページのURLが「/service/」「/about/」「/contact/」という形式で機能するためには、WordPressのパーマリンク設定を変更する必要があります。
デフォルトの設定では「/?page_id=123」という読みにくいURLになってしまいます。

パーマリンクの設定手順
  • 【STEP 1】左メニューの「設定」→「パーマリンク設定」をクリック
  • 【STEP 2】「投稿名」のラジオボタンをクリックして選択する
  • → これでURLが「https://サイトURL/ページ名/」という読みやすい形式になる
  • → サンプルURLが「https://example.com/sample-post/」のように変わればOK
  • 【STEP 3】「変更を保存」ボタンをクリックする
  • → この設定をしてから各固定ページのスラッグを設定すると
  • 　「https://サイトURL/service/」「https://サイトURL/about/」という
  • 　URLが正しく機能するようになる

第4章：CSSとJavaScriptをWordPressで読み込む
──デザインを正しく表示させる
4-1. なぜHTMLを貼り付けただけではデザインが崩れるのか
ClaudeのHTMLをWordPressの固定ページに貼り付けてプレビューすると、最初はデザインが崩れた状態に見えることがあります。
これはTailwind CSSのCDNリンクとGoogle FontsのリンクがClaudeのHTMLの<head>部分にあったのに、<body>部分だけを貼り付けたためです。
<head>の内容が読み込まれていないのでCSSが効かず、フォントも適用されていない状態になります。
この問題を解決するために、TailwindとGoogle FontsをWordPress側で全ページに読み込む設定を行います。方法は主に2つあります。
①functions.phpに追記する方法（推奨）と、②外観→カスタマイズの「追加HTML」に追記する方法です。①は正しいWordPressの方法でより安定しており、②は手軽ですが一部テーマでは機能しない場合があります。
初心者には②を試してから①に進む順番がおすすめです。
4-2. 方法①：外観→カスタマイズから読み込む（簡単・まず試す方法）

🖥️ 外観→カスタマイズでCDNを追加する手順
  • 【STEP 1】左メニューの「外観」→「カスタマイズ」をクリック
  • → フロントエンドのプレビューが付いたカスタマイズ画面が開く
  • 【STEP 2】左側のメニューから「追加CSS」を探してクリック
Error uploading image.
  • ※テーマによっては「追加JavaScript」や「カスタムHTML」のような項目がある場合もある
  • 【STEP 3】以下のコードを入力欄に貼り付ける 
@import url("https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap");
body { font-family: "Noto Sans JP", sans-serif; }
  • ※ただしここはCSSのみ対応。TailwindのJSは別途設定が必要
  • 【STEP 4】「公開」ボタンをクリックして保存する
【TailwindのCDN（JavaScript）を追加する場合】
  • → テーマによって「外観→カスタマイズ→追加スクリプト」や「外観→ウィジェット→フッターウィジェット」などから追加できる場合がある
  • → テーマが対応していない場合は方法②のfunctions.phpを使う
4-3. 方法②：functions.phpに追記する（確実・推奨）
functions.phpはWordPressテーマのメインPHPファイルで、ここにコードを追加することでサイトの全ページに任意のスクリプトを読み込ませることができます。
これがWordPressの正しい実装方法です。
ただし誤ったコードを追加するとサイトが真っ白になることがあるため、必ずバックアップを取ってから行ってください。

functions.phpを編集する前に必ずやること
  • 1. 現在のfunctions.phpの全内容をコピーしてテキストファイルとして手元に保存しておく
  • 2. 保存場所：デスクトップの「functions_php_バックアップ_日付.txt」など
→ もしサイトが真っ白になったら、このバックアップを使ってFTPソフト（FileZilla）で元のファイルに上書きして復旧できる

🖥️ functions.phpへのコード追加手順
  • 【STEP 1】左メニューの「外観」→「テーマファイルエディター」をクリック
  • → 画面右側に「テーマファイルの編集」というリストが表示される
  • 【STEP 2】右側のファイルリストから「functions.php」を探してクリック
  • → ファイルの内容がエディター画面に表示される
  • 【STEP 3】ファイルの内容の「最後の？>の直前」または「末尾」に以下のコードを追加する（Claudeに生成させると確実） 
function add_tailwind_and_fonts() { 　wp_enqueue_script("tailwindcss", 　　"https://cdn.tailwindcss.com", array(), null, false); 　wp_enqueue_style("noto-sans-jp", 　　"https://fonts.googleapis.com/css2?family=Noto+Sans+JP:wght@400;700&display=swap", 　　array(), null); } add_action("wp_enqueue_scripts", "add_tailwind_and_fonts");
  • 【STEP 4】「ファイルを更新」ボタンをクリックして保存
  • 【STEP 5】サイトをブラウザで開いてデザインが正しく表示されることを確認

4-4. ナビゲーションのリンクをWordPress形式に変更する

5-1. Contact Form 7でお問い合わせフォームを実装する
8-4・8-5でHTMLのお問い合わせフォームを作成しましたが、このHTMLフォームだけでは実際にメールが届くお問い合わせ機能を持っていません。

🖥️ Contact Form 7のインストールから設定まで完全手順
  • 【STEP 1】プラグインのインストール
  • → 左メニューの「プラグイン」→「プラグインを追加」をクリック

  • → インストール完了後「有効化」ボタンをクリック
  • → 左サイドバーに「お問い合わせ」メニューが追加されればインストール完了
  • 【STEP 2】デフォルトフォームの確認と編集

  • → フォームの「フォーム」タブで項目を確認・編集できる
  • → デフォルトは「氏名・メール・件名・本文」の4項目が設定済み
  • → 必要に応じて「会社名」「電話番号」等の項目を追加できる
  • 【STEP 3】受信先メールアドレスの設定（重要）
  • → 「メール」タブをクリック

  • → フォームの編集画面の最上部に「このショートコードをコピーして固定ページに貼り付けてください」というショートコードが表示される

  • → このショートコードをコピーする
  • 【STEP 5】固定ページにショートコードを貼り付ける
  • → contact.html組み込み済みの固定ページを開く

  • → 公開されたお問い合わせページからテスト送信する
  • → クライアントのメールアドレスに届くか確認する
  • → 届かない場合：「送信先」メールアドレスの設定ミスやサーバーのメール送信設定を確認する
