1-2. 「指示の精度」がHTML品質を決めるという事実

LPの各セクションは、この感情の流れの各段階に対応しています。つまりフェーズ3を学んだあなたは、LPの設計図をすでに頭の中に持っています。

2-2. 「内なる独り言を代弁する」技術がFVを決める
LP制作で最も重要でかつ最も差が出るのが、FVのキャッチコピーです。
フェーズ3で徹底的に学んだ「ターゲットの内なる独り言を代弁する」技術は、このFV設計において最大の威力を発揮します。具体的な比較を見てください。

同一ターゲット・同一商品でのFVキャッチコピー比較
【商品・ターゲット】副業支援のオンラインスクール。対象：本業があり副業を始めたいが何から手をつければいいかわからない30代サラリーマン
【パターンA：機能訴求型（デザイナー・エンジニアがよく作るタイプ）】
「副業で月10万円を実現するオンラインスクール。実績多数・初心者歓迎」
→ 情報は正確。しかし「自分のための言葉」とは感じにくい
→ 「また同じようなコピーだ」という印象を与えるリスク
【パターンB：問いかけ型（よくある手法）】
「副業に興味はあるけど、何から始めればいいかわからない方へ」
→ ターゲットに向けている点は良い。ただし「よくある言い回し」で刺さりが弱い
【パターンC：内なる独り言を代弁（フェーズ3の技術）】
「なんとなく副業を調べ始めて、もう3ヶ月が経った。」
→ ターゲットがまさに今感じている「焦りと停滞感」を言語化している
→ 読んだ瞬間に「これ、私のことだ」と感じさせる
→ 続きを読みたくなる「引力」が生まれる
→ パターンCを書けるのは、ライティングを学んだ人だけ
→ Claudeに「ターゲットの内なる独り言を代弁するキャッチコピーを書いて」と指示できる人だけ
2-3. 感情設計をClaudeへの指示に落とし込む方法
感情設計の概念を理解しているだけでは不十分です。
それをClaudeへの指示として正確に言語化できて初めて、高品質なLPが生成されます。ライター出身の制作者が非ライターと決定的に違うのはここです。
「感情設計を重視して書いて」という抽象的な指示では、Claudeも最大限のパフォーマンスを発揮できません。以下のように、感情の流れを具体的に指定することが重要です。

感情設計をClaudeへのLP指示に変換するテンプレート
「以下の感情設計に沿って、LPの本文を書いてください。
【ターゲットの現状感情（スタート地点）】
〇〇に悩んでいる。△△を試したが続かなかった。□□という言葉が頭の中でぐるぐるしている。
【各セクションで引き起こしたい感情の変化】
・FVキャッチ：「これは自分のことだ」→続きを読みたくなる
・問題提起：「そうそう、まさにこれ」という強い共感
・解決策：「この方法なら自分にもできそう」という希望
・実績：「本当に効果があるんだ」という信頼
・CTA：「今すぐやらないと損をする」という動機づけ
【この感情の流れを邪魔してはいけないもの】
・難しい専門用語（感情の流れを止める）
・過剰な断定表現（不信感につながる）
・説明的・説教的なトーン（「共感」から「指導」に変わってしまう）
各セクションの末尾に「次のセクションへの橋渡し文」を必ず入れてください」
2-4. 「ペルソナ設計」がコードのクオリティを上げる具体的な理由
フェーズ3でペルソナ設計（ターゲットの詳細な人物像を作ること）を学びました。
これはLP・HP制作においてClaudeへの指示精度を上げる重要な手順です。
ペルソナを詳細に設定してからClaudeに指示すると、HTMLの中のコピーの精度が大幅に上がります。なぜなら、Claudeはペルソナの情報を受け取ることで「この人に向けた言語はどんなものか」という推論ができるようになるからです。
たとえば「30代の女性に向けたダイエットサプリのLP」と指示するより
「35歳・2人の子どもがいる・育児と在宅ワークで忙しい・産後から10kg増えていて4回のダイエットに失敗した・朝30分だけ自分の時間がある・InstagramでダイエットアカウントをフォローしているがTipsを試す気力がない」
と指示した方が、出力されるHTMLのコピーの精度が格段に上がります。ペルソナの深さはそのままコードの質になります。

ペルソナ設計からClaudeへの指示の「翻訳テンプレート」
【STEP 1】フェーズ3の手法でペルソナを詳細に設計する
　→ 年齢・性別・職業・家族構成・収入・悩みの深さ・失敗経験・欲求を言語化
【STEP 2】ペルソナを「ターゲットの状態の言語化」に変換する
　→「今このターゲットが心の中で使っている言葉は何か」を1〜2文で書く
　　例：「また今週も運動できなかった。もう自分には無理なのかも。でも…」
【STEP 3】ClaudeへのLP指示の冒頭にペルソナと状態言語化を貼り付ける
　→「以下のターゲット情報をもとにLPを書いてください。
　　 特にFVのキャッチコピーはターゲットの「状態言語化」を代弁する表現で書いて」
→ このSTEP 1〜3を踏むだけで、Claudeの出力精度が段違いに上がる
→ フェーズ3を学んだあなたは、STEP 1を他の制作者より深く実行できる
第3章：フェーズ4「ブランドボイスGL」がHP制作のすべてを決める
3-1. ブランドボイスGLとはWebの「世界観設計書」のこと
フェーズ4で学んだブランドボイスガイドライン（BV-GL）とは、「このブランドがすべてのコンテンツで守るべきトーン・言葉遣い・感情的な方向性の設計書」でした。HP制作において、このBV-GLはClaudeへの全ページ指示の核心的な素材になります。
HPは複数のページで構成されます。
トップページ・サービス紹介・会社概要・お問い合わせ・ブログ等、ページが増えるほど「一貫した世界観を維持する」ことが難しくなります。
デザイナーやエンジニアがHTMLを生成する場合、各ページを個別に設計することが多く、「トップページは温かみがあるのにサービスページは無機質」という世界観のブレが生じやすい。
ライター出身の制作者がBV-GLを先に設計してからClaudeに指示すれば、全ページを通じた統一感が最初から確保されます。
3-2. BV-GLをClaudeへのHP生成指示に変換する方法
フェーズ4で作ったBV-GLの項目は、そのままClaudeへのHP生成指示に転用できます。BV-GLがあれば「このGLに沿ってHPのトップページのHTMLを書いて」という指示だけで、世界観が統一されたコードが生成されます。これはBV-GLを持っていない制作者には不可能な指示です。
BV-GLの項目
HP制作での使い方
Claudeへの指示への変換例
ブランドパーソナリティ（形容詞3つ）
HTML内のコピー・CTA文言・サービス名称の表現に反映
「このブランドは「温かく・正直で・専門的」なパーソナリティです。すべてのコピーはこの3つの形容詞を体現する言葉で書いてください」

「口調は「ですます」調。一文は60字以内。「〜と思います」は使わない。読者に語りかける「あなた」を使う」

3-3. BV-GLを先に作ることで「制作後の修正」が激減する理由
HP制作でクライアントから最も多い修正依頼は「なんか違う」というトーンの問題です。
デザインは良いのに、コピーの口調がブランドに合っていない。
そういった感覚的な修正が繰り返されることで、制作期間が延びます。BV-GLを最初に設計してクライアントの確認を取ってから制作を始めれば、この問題が根本的になくなります。「GLに沿って制作しています」という説明ができるため、クライアントも「方向性がずれている」という感覚を持ちにくくなるのです。
さらに重要なのは、BV-GLをクライアントと作成するプロセス自体が「ヒアリングの深度」を上げるということです。「御社のブランドを3つの形容詞で表すと？」「使いたい言葉・使いたくない言葉は？」という問いは、デザイナーやエンジニアが通常しない質問です。このプロセスを経ることで、クライアントはあなたに「ブランドを本当に理解してくれている」という印象を持ちます。これが単価交渉における差別化につながります。

最後に「このGLを使ってClaude自身が最初に生成するトップページのFVコピーのサンプル（50字）」を作ってください」
第4章：フェーズ7-3「Claude攻略」がコード生成を制する
4-1. プロンプト設計技術はそのままコード生成指示に転用できる
上のグラフは「非ライターの制作プロセス」と「ライター×AIの制作プロセス」を各工程の深さで比較しています。
非ライターのプロセスでは各工程が2〜3点なのに対して、ライター×AIは5点満点が並んでいます。この差を生んでいるのは「各工程でどれだけ深い言語化ができているか」です。
フェーズ7-3でClaudeへのプロンプト設計を学んだとき、「具体的な指示ほど品質が上がる」「ネガティブ指示（NGリスト）が有効」「最初に全情報を詰め込む」というポイントを学びました。
これらはコード生成の指示にも完全に当てはまります。「ライティングのプロンプト設計＝コード生成のプロンプト設計」という等式が成り立ちます。

4-3. 「一発で動くHTML」を生成するためのライター式プロンプト
「Claudeにコードを生成させるとエラーが出る」「修正が多くて大変」という話を聞いたことがあるかもしれません。
これはほぼすべてプロンプトの書き方の問題です。情報が少なく曖昧な指示を出すと、Claudeは「一般的なLP」を生成し、後から「そうじゃない」という修正が大量に発生します。
フェーズ7-3で学んだプロンプト設計の「最初に全情報を詰め込む」アプローチを使えば、この問題がほぼなくなります。

ライター式「一発で高品質なLP HTML」を生成するプロンプト構造
【構成要素①：制作の前提を定める】
「以下の情報をもとに、LP全体のHTMLとCSSを生成してください。
使用技術：HTML5 ＋ Tailwind CSS（CDN使用）
要件：モバイルファーストのレスポンシブ対応」
【構成要素②：ターゲットとBVGLを渡す】
「ターゲット：[詳細なペルソナ]
ブランドボイス：[3つの形容詞と意味]
NG表現：[避けるべき言葉のリスト]」
【構成要素③：感情設計を構造で渡す】
「LP構成：FV（感情目標：共感）→問題提起（感情目標：共感強化）→
解決策（感情目標：希望）→特徴×3（感情目標：信頼）→CTA（感情目標：行動）
各セクションで達成すべき感情の変化も意識してコピーを書いてください」

4-4. 「会話で修正する」という制作スタイルの定着
Claudeへのコード生成の最大の強みの1つは「会話で修正できる」ことです。フェーズ7-3でClaudeに長文を修正させるとき「この部分をもう少し感情的にして」「ここの表現をもっとシンプルに」という自然言語での修正指示に慣れていたはずです。HTMLの修正も全く同じです。
「このヘッダーの背景色を少し暗くして」「CTAボタンの角丸をもう少し大きくして」「このセクションの文字サイズを一回り大きくして」という指示が、そのままコード修正として機能します。
