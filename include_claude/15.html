1-5. Claudeに「狙った文章」を書かせる指示の出し方

フェーズ1｜Claude×ライティング基礎構築編

はじめに：プロンプトが9割

ここまで、ライティングの「型」「読まれる文章の特徴」「マーケティングフレームワーク」を学んできました。

この章では、それらの知識をClaudeに「正確に伝える」技術を学びます。

結論から言います。Claudeの出力品質の9割は、プロンプト（指示文）で決まります。

同じClaudeを使っても、プロンプト次第で「使えない文章」にも「即納品できる文章」にもなります。この差を生むのが、「プロンプトエンジニアリング」のスキルです。

【プロンプトの違いによる出力の差】

【素人のプロンプト】「ダイエットサプリの記事を書いて」

→ Claudeの出力：「ダイエットサプリは、健康的な体重管理をサポートするために設計された栄養補助食品です。市場には様々な種類のサプリメントがあり、それぞれが異なるアプローチで…」

→ 一般的で誰にも刺さらない文章。そのまま使えない。

【プロのプロンプト】

「あなたはセールスライティングのプロです。以下の条件で、ダイエットサプリの紹介文を書いてください。

  • ■ターゲット：40代女性、働いている、運動する時間がない
  • ■構成：PASONAの法則で書く
  • ■各パートの内容：P：代謝低下による体重増加の悩み / A：同じ悩みを持っていた経験を交える / S：代謝アップ成分の効果と科学的根拠 / O：初回50%OFF、30日返金保証 / N：先着100名限定 / A：今すぐ申込ボタンをクリック
  • ■トーン：共感的で、押しつけがましくない
  • ■文字数：800文字程度」

→ Claudeの出力：ターゲットに刺さる、構成が整った、すぐに使える文章が出てくる。

この差は、才能やセンスではありません。「プロンプトの書き方」を知っているかどうかの差です。

この章を学び終える頃には、あなたはClaudeから「狙い通りの文章」を引き出せるようになります。

第1章：プロンプトの基本構造

良いプロンプトには、共通する「構造」があります。この構造を理解すれば、どんな文章でも「狙い通り」に出力させられます。

1-1. プロンプトの5つの要素

プロンプトは、以下の5つの要素で構成します。

【プロンプトの5要素】

① 役割（Role）：Claudeに「誰として」書くかを指定する。「あなたは〇〇のプロです」

② 文脈（Context）：背景情報を提供する。ターゲット、目的、商品情報など。

③ 指示（Instruction）：何をしてほしいかを明確に伝える。「〇〇を書いてください」

④ 形式（Format）：出力の形式を指定する。構成、文字数、トーンなど。

⑤ 例示（Example）：具体例を示す。「こんな感じで」「この文章を参考に」

これらの要素をすべて含める必要はありませんが、含める要素が多いほど、出力の精度は上がります。

1-2. 各要素の詳細と書き方

① 役割（Role）の設定

Claudeに「専門家の視点」で書かせるために、役割を設定します。役割を設定することで、専門的で質の高い出力が得られます。

【役割設定の例】

「あなたはセールスライティングのプロです。」

「あなたは10年の経験を持つWebライターです。」

「あなたはマーケティングコンサルタントです。」

「あなたは40代女性向け美容メディアの編集長です。」

「あなたは読者の心を動かすコピーライターです。」

【役割設定がない場合の問題】

役割設定なしのプロンプト：「ダイエットサプリの記事を書いて」

→ Claudeは「誰として」書けばいいかわからない → 一般的な説明文になりがち

役割設定ありのプロンプト：「あなたはセールスライティングのプロです。ダイエットサプリの記事を書いてください」

→「セールス」の視点で、購買を促す文章を書く

② 文脈（Context）の提供

Claudeに「前提情報」を与えることで、的確な出力が得られます。

【文脈として伝えるべき情報】

  • ターゲット：誰に向けた文章か
  • 目的：何のための文章か（販売、認知、教育など）
  • 商品・サービス情報：特徴、価格、競合との違い
  • 媒体：どこに掲載するか（Web、SNS、メルマガなど）
  • 読者の状態：すでに興味があるか、まだ認知していないか

【文脈提供の例】

  • ■ターゲット：40代女性 / 働いている（フルタイム）/ 運動する時間がない / 過去にダイエットに失敗した経験がある
  • ■目的：商品購入につなげるLP用の文章
  • ■商品情報：商品名：スリムアップサポート / 特徴：代謝アップ成分配合、1日1粒でOK / 価格：月額3,000円（初回1,500円）/ 強み：40代の体に特化した成分設計
  • ■掲載媒体：公式サイトのランディングページ

③ 指示（Instruction）の明確化

「何をしてほしいか」を具体的に伝えます。

【曖昧な指示の問題】

曖昧な指示：「いい感じの記事を書いて」「読者に刺さる文章にして」「もっと良くして」

→ Claudeは「何をすればいいか」わからない → 期待と違う出力が返ってくる

【具体的な指示の例】

「PASONAの法則に従って、導入文を書いてください」

「Problem部分を、より共感を呼ぶ表現にリライトしてください」

「冒頭の3行を、問いかけ型で書き直してください」

「ベネフィットを3つ追加してください」

「一文を50文字以内に分割してください」

指示は「動詞」で終わるように書くと明確になります。「〇〇してください」「〇〇を書いてください」

④ 形式（Format）の指定

出力の「形式」を指定することで、意図通りの出力が得られます。

【形式で指定できること】

  • 構成：PASONA、PREP、AIDMAなどの型
  • 文字数：「800文字程度」「500〜700文字」
  • 段落数：「3段落で」「見出し4つで」
  • トーン：「カジュアル」「フォーマル」「共感的」
  • 語尾：「です・ます調」「だ・である調」
  • 禁止事項：「専門用語を使わない」「箇条書きは使わない」

【形式指定の例】

  • ■構成：PASONAの法則で書く
  • ■文字数：800文字程度（±100文字は許容）
  • ■トーン：共感的で、優しい / 押しつけがましくない / 読者と同じ目線で語りかける
  • ■語尾：です・ます調
  • ■禁止事項：専門用語を使わない / 「絶対」「必ず」などの断言表現を避ける / 価格を強調しすぎない

⑤ 例示（Example）の活用

具体例を示すことで、Claudeの理解度が大幅に上がります。

【例示の方法】

  • 良い例を示す：「このような文章を書いてください」
  • 悪い例を示す：「このような文章は避けてください」
  • 参考文章を示す：「この文章のトーンを参考にしてください」
  • フォーマットを示す：「出力はこの形式で」

【例示を含むプロンプト例】

以下の「良い例」を参考に、導入文を書いてください。

【良い例】「40歳を過ぎてから、何をしても痩せなくなった…」もし、あなたがそう感じているなら、この記事はあなたのために書きました。実は、40代のダイエットには、20代・30代とは全く違う「コツ」があるのです。

【良い点】ターゲットの悩みを具体的に描写している / 「あなたのため」と個人に語りかけている / 「コツがある」と続きを読みたくなる仕掛けがある

この要素を含んだ導入文を書いてください。

第2章：プロンプトテンプレート集

ここからは、すぐに使えるプロンプトテンプレートを紹介します。これらをベースに、案件に合わせてカスタマイズしてください。

第3章：出力を改善する修正テクニック

Claudeの出力が期待通りでない場合、適切な修正指示で改善できます。この章では、よくある問題と修正方法を解説します。

3-1. 問題別の修正プロンプト

【問題1：内容が一般的すぎる】

【修正プロンプト】

この文章は一般的すぎます。以下の点を改善してください。

1. ターゲット（[具体的なターゲット]）に特化した内容にする

2. 具体的な数字やデータを追加する

3. [ターゲット]ならではの悩みや状況を描写する

特に冒頭を「読者が自分のことだと感じる」内容に書き換えてください。

【問題2：トーンが合っていない】

【修正プロンプト】

この文章のトーンを調整してください。

【現状】堅い、説明調、距離感がある

【目標】親しみやすい、共感的、友達に話すような

具体的には：読者に語りかける表現を増やす / 「〜である」→「〜なんです」/ 短い文を混ぜてリズムを良く

内容は変えず、トーンだけを調整してください。

【問題3：構成が崩れている】

【修正プロンプト】

この文章をPASONAの構成に整理し直してください。

現状の文章を以下の形式で再構成してください：

【P：Problem】（300文字）/ 【A：Affinity】（150文字）/ 【S：Solution】（300文字）/ 【O：Offer】（150文字）/ 【N：Narrowing】（100文字）/ 【A：Action】（100文字）

各パートの内容は現状から抽出し、足りない部分は追加してください。

【問題4：文字数が合わない】

【短縮する場合】

この文章を[目標文字数]文字以内に短縮してください。

  • ■残すべき内容（優先度順）：1. ターゲットの悩みへの共感 / 2. 主要ベネフィット / 3. CTA
  • ■削ってよい内容：詳細な説明 / 重複している部分 / 冗長な表現

【拡張する場合】

この文章を[目標文字数]文字程度に拡張してください。

  • ■追加してほしい要素：具体例やエピソード / 読者の悩みの詳細な描写 / ベネフィットの深掘り
  • ■注意：水増しではなく、価値ある情報を追加 / 全体のバランスを維持

3-2. 段階的改善のフレームワーク

一度で完璧を求めず、段階的に改善するアプローチが効果的です。

【4段階改善フレームワーク】

Step 1：全体を出力させる　まず大まかな指示で全体像を出力。細かい調整は後で。

Step 2：構成を確認・修正　流れ、各パートの役割が正しいか確認。問題があれば修正指示。

Step 3：内容を改善　各パートの内容をチェック。具体性、説得力を高める。

Step 4：表現を磨く　トーン、言葉選び、リズムを調整。最終仕上げ。

【実践例】

【Step 1】「ダイエットサプリの紹介文をPASONAで800文字で書いてください」

【Step 2】「構成を確認しました。Problem部分が弱いので、もっと具体的な悩みの描写を追加してください」

【Step 3】「Solution部分に、なぜ代謝アップが効果的なのか、理由を1つ追加してください」

【Step 4】「全体のトーンをもう少しカジュアルに。友達に勧めるような温かみを出してください」

第4章：プロンプトを資産化する

良いプロンプトは、何度も使える「資産」です。効率的に再利用するための整理・保存方法を解説します。

4-1. プロンプトライブラリの作り方

よく使うプロンプトをライブラリとして整理しましょう。

【ライブラリの分類例】

プロンプトライブラリ/

├── セールス系/ PASONA_LP用.txt / AIDMA_認知向け.txt / 商品説明_EC用.txt

├── 記事系/ SEO記事_標準.txt / ハウツー記事.txt / 比較記事.txt

├── SNS系/ Twitter投稿.txt / Instagram投稿.txt / LINE配信.txt

├── メール系/ メルマガ_情報提供.txt / メルマガ_セール告知.txt

└── リライト系/ 共感強化.txt / 具体化.txt / 短縮.txt

4-2. プロンプトの変数化

プロンプトの「変わる部分」を変数化すると、使い回しが楽になります。

【変数化の例】

あなたは{役割}です。

{商品名}について、{構成}の法則で紹介文を書いてください。

  • ■ターゲット {ターゲット詳細}
  • ■商品情報：商品名：{商品名} / 特徴：{特徴} / 価格：{価格}
  • ■トーン：{トーン}
  • ■文字数：{文字数}文字程度

---【使用時の記入例】

{役割}=セールスコピーライター / {商品名}=スリムサポートサプリ / {構成}=PASONA

{ターゲット詳細}=40代女性、働いている、運動時間がない

{トーン}=共感的、押しつけがましくない / {文字数}=800

第5章：よくある失敗と対策

プロンプトでよくある失敗パターンと対策を紹介します。

失敗1：指示が曖昧

【失敗例】

「もっと良くして」「いい感じに」「刺さる文章に」→ 何を改善すべきかわからない

【対策】

「何を」「どのように」を具体的に。

×「もっと良くして」　○「冒頭3行を、問いかけ型に書き換えてください」

×「刺さる文章に」　○「40代女性が『自分のことだ』と感じる悩みの描写を追加」

失敗2：情報の詰め込みすぎ

【失敗例】

ターゲットの情報を文章でダラダラ書く → Claudeが混乱し、焦点がぼやける

【対策】

情報は箇条書きで整理。優先順位をつける。

失敗3：一度で完璧を求める

【失敗例】

「完璧な記事を一発で書いて」→ 無難な出力になる。修正しづらい

【対策】

段階的に進める。構成→各パート→仕上げ。

失敗4：出力をそのまま使う

【失敗例】

Claudeの出力をそのままコピペ納品 → 「AIっぽい」文章。クライアントに見抜かれる

【対策】

出力は「下書き」。必ず人間がチェック・編集する。

【チェックポイント】

  • ターゲットに刺さる具体性があるか
  • 「AIっぽい」表現が残っていないか
  • 情報は正確か
  • クライアントのトーンに合っているか

【課題】

5. LP用（PASONA構成）

まとめ：プロンプトは「第二の文章力」

この章では、Claudeから「狙った文章」を引き出すプロンプト技術を学びました。

重要なポイントを振り返りましょう。

【プロンプトの5要素】

  • 役割（Role）：誰として書くか
  • 文脈（Context）：背景情報を提供
  • 指示（Instruction）：何をしてほしいか
  • 形式（Format）：出力の形式を指定
  • 例示（Example）：具体例を示す

【修正テクニック】

  • 問題点を具体的に指摘する
  • 「何を」「どのように」を明確に
  • 段階的に改善する
  • 出力は下書きとして扱う

【プロンプトの資産化】

  • ライブラリとして整理
  • 変数化で再利用しやすく
  • 改善記録をつける

「良い文章を書く力」と「良いプロンプトを書く力」は、これからのライターの両輪です。

プロンプト力は、「第二の文章力」と言っても過言ではありません。
