フェーズ0｜思考OSのインストール編

0-3. 画像・動画・自動化・SNSの前に必ず存在する工程

前章では、すべてのAI副業に「言語設計」という共通の土台があることを学びました。しかし、多くの人はこの重要な工程を「見落としている」のが現実です。

なぜ見落とすのか。それは、この工程が「目に見えにくい」からです。画像生成ツールを開けば、すぐに画像が作れます。動画編集ソフトを立ち上げれば、すぐに編集を始められます。SNSアカウントを開設すれば、すぐに投稿できます。

その「すぐに」の前に、必ず存在する工程があります。それが「言語による設計」です。

この章では、画像・動画・自動化・SNSという4つの分野を取り上げ、それぞれの実際の作業フローを詳細に分解します。そして、その中で必ず存在する「言語設計の工程」を可視化していきます。

この理解があることで、「なぜライティングを先に学ぶ必要があるのか」が、理論ではなく実感として理解できるようになります。

【工程6】クライアントへの説明（100%言語作業）

最終的に生成した画像をクライアントに提出する際も、言語によるコミュニケーションが必要です。単に画像を送るだけでなく、以下のような説明を行います：

  • 「ご要望の『爽やかで映える』という方向性を表現するために、自然光を活かした明るいトーンにしました」
  • 「フルーツの色味を鮮やかに仕上げることで、Instagramでの視認性を高めています」
  • 「ポスター用に縦型・横型の2バージョンを用意しました。どちらをご希望ですか？」

この説明ができるかどうかが、プロとアマチュアの大きな違いです。画像の意図を言語で説明できることで、クライアントの信頼を得ることができます。

1-2. 全工程を振り返る：実は何割が言語作業か？

ここまでの工程を整理すると、画像生成案件における作業の内訳が見えてきます。

  1. 工程1：要件ヒアリング　→　100%言語作業
  2. 工程2：コンセプト設計　→　100%言語作業
  3. 工程3：プロンプト作成　→　100%言語作業
  4. 工程4：画像生成・評価　→　80%が言語による判断作業
  5. 工程5：プロンプト修正・再生成　→　100%言語作業
  6. 工程6：クライアントへの説明　→　100%言語作業

全工程を通じると、約95%が言語による作業です。「画像生成AI」と呼ばれていても、実際の業務の大半は「言語設計」なのです。これが、画像生成において文章力が最も重要なスキルである理由です。

1-3. 個人制作の場合でも同じ構造がある

「クライアント案件ではなく、個人で趣味として使う場合はどうだろう？」と思う方もいるかもしれません。しかし、構造は変わりません。たとえば「自分のSNSに載せる風景画像を作りたい」という場合でも、頭の中では次のような言語思考が行われています。

「夕焼けの海が見たい。でもありきたりじゃなくて、少し幻想的な感じがいい。光の色は暖色系で、空にはうっすら雲がある。人物は入れずに、静けさを表現したい……」

この思考を正確にプロンプトへ変換できるかどうかが、生成される画像の質を決定します。言語で明確に「設計」できる人ほど、意図通りの画像を生み出すことができるのです。

1-2. 全工程における言語作業の割合

上記の工程を時間配分で見ると、以下のようになります。

  1. 工程1（要件ヒアリング）：30分
  2. 工程2（コンセプト設計）：20分
  3. 工程3（プロンプト作成）：15分
  4. 工程4（画像生成と評価）：10分
  5. 工程5（修正と再生成）：20分
  6. 工程6（説明文作成）：15分

合計：110分

このうち、AIが画像を生成している時間は実質5分程度です。残りの105分（約95%）は、言語を使った思考・設計・コミュニケーションに費やされています。

「画像生成」という仕事の本質は、画像を作ることではなく、言語で設計することなのです。

また、完成した画像をクライアントに提出する際には、必ず「説明」が必要です。その説明の質が、クライアントの満足度と継続依頼に直結します。

「お待たせいたしました。夏限定フルーツパフェの告知画像を作成いたしました。ポイントは以下の3つです：①自然光を活かした明るい雰囲気で爽やかさを表現、②フルーツの鮮やかな色を引き立てるために背景はパステルトーンに統一、③Instagramで映えるよう接写でパフェの美しさを強調。ターゲット層の20〜30代女性が「食べたい！」と思える仕上がりを目指しました。ご確認いただき、修正点があればお知らせください。」

【工程5】編集作業（50%言語作業、50%技術作業）

素材が揃ったら、実際の編集作業に入ります。この工程は技術的な操作が伴いますが、判断の半分は依然として言語的思考に基づいています。

編集作業における言語的判断

  • テロップの文言・フォント・表示タイミングの決定
  • ナレーション原稿の読み上げ速度と映像の整合性確認
  • BGMの音量バランスと台本のトーンとの一致
  • カット割りのテンポが構成表の意図通りかのチェック

編集ソフトを操作するのは技術スキルですが、「何をどの順番で、どんな言葉を添えて見せるか」を決めるのは、企画・構成・台本で積み上げてきた言語設計の力です。

動画制作フロー全体のまとめ

  1. 企画立案（100%言語作業）：動画の目的・ターゲット・メッセージを言語で定義する
  2. 構成設計（100%言語作業）：タイムライン上に言葉で骨格を組み立てる
  3. 台本作成（100%言語作業）：映像・音声・テロップを統合した設計図を書く
  4. 素材指示書作成（100%言語作業）：AIや外注先への精密な言語指示を行う
  5. 編集作業（50%言語作業・50%技術作業）：言語設計を映像として具現化する

「動画制作＝映像スキル」と思われがちですが、実際には工程全体の大半を言語作業が占めています。ライティング力のある人ほど、質の高い動画を効率よく作れるのです。

編集・レビュー工程における言語思考

実際の編集作業における言語思考の例

編集ソフトを操作するのは技術作業ですが、「何をどう調整するか」の判断は、すべて言語で行われます。

  • 「このシーンは5秒だと長すぎる。3秒に短縮しよう」（言語思考）
  • 「ナレーションと映像のタイミングがずれている。0.5秒遅らせよう」（言語思考）
  • 「テロップが読みづらい。フォントサイズを大きくしよう」（言語思考）

【工程6】レビューと修正（100%言語作業）

完成した動画をレビューし、修正点を洗い出します。これらすべて、言語で思考し、言語で判断します。

【レビューチェックリスト】

  1. メッセージは明確に伝わっているか？
  2. ターゲット層に刺さる内容か？
  3. 視聴者が最後まで見たくなる構成か？
  4. 購入を促す要素は十分か？
  5. ナレーションのトーンは適切か？
  6. テロップは読みやすいか？

動画制作のすべての工程において、判断・評価・改善のプロセスは一貫して言語思考によって支えられている。

2-2. 全工程における言語作業の割合

60秒の動画を作成する場合の時間配分：

  1. 工程1（企画立案）：30分
  2. 工程2（構成設計）：20分
  3. 工程3（台本作成）：60分
  4. 工程4（素材指示書作成）：20分
  5. 工程5（編集作業）：90分
  6. 工程6（レビューと修正）：30分

合計：250分（約4時間）

このうち、純粋な「映像編集技術」を使っている時間は90分の半分、つまり45分程度です。残りの205分（約82%）は、言語を使った企画・設計・台本作成・判断です。

つまり、動画制作の本質も、言語設計なのです。

第3章：SNS運用の実際のフロー

SNS運用は「投稿するだけ」ではなく、綿密な言語設計が必要です。美容サロンのInstagram運用代行（月20投稿）を例に、全工程を見ていきましょう。

3-1. Instagram運用代行の作業フロー

【工程1】アカウント分析と戦略立案（100%言語作業）

【分析レポート例】
現状：フォロワー500名、エンゲージメント率1.2%
課題：投稿内容が施術写真のみで、ストーリー性がない
目標：3ヶ月でフォロワー1,500名、エンゲージメント率3%以上
戦略：Before/After・お客様の声・美容Tips・スタッフ紹介を組み合わせる

この分析と戦略立案は、完全に言語で行われます。

【工程2】コンテンツカレンダー作成（100%言語作業）

1ヶ月の投稿計画を言語で設計します。

【3月のコンテンツカレンダー（抜粋）】
3/1（月）：春の新メニュー紹介
3/3（水）：お客様のBefore/After
3/5（金）：美容Tips「春の肌荒れ対策」
3/8（月）：スタッフ紹介（エステティシャンA）

【工程3】個別投稿の企画（100%言語作業）

各投稿について、詳細を言語で設計します。

【投稿企画書：3/5「春の肌荒れ対策」】
目的：フォロワーに価値を提供し、エンゲージメントを高める
内容：春に多い肌荒れの原因と対策を3つ紹介
画像：清潔感のある肌のクローズアップ写真
キャプション：共感→問題提起→解決策→行動喚起の流れ
ハッシュタグ：#美容 #スキンケア #春 #肌荒れ対策 など10個

【工程4】キャプション作成（100%言語作業）

最も重要なのがキャプション（投稿文）の作成です。優れたキャプションには以下の要素が含まれます。

  • 共感を呼ぶ書き出し：読者が「あるある！」と感じる冒頭文
  • 読者の興味を引く問題提起：「なぜ春に肌荒れが増えるのか？」
  • 具体的で実用的な情報：対策3選など、保存したくなるコンテンツ
  • 絵文字による視覚的な強調：読みやすさと親しみやすさを演出
  • 自然な行動喚起：「プロフィールのリンクからご予約📝」
  • 適切なハッシュタグ：リーチを最大化する10〜15個の選定

「春になると肌が荒れる...😢 そんなお悩み、ありませんか？ 実は春は、一年で最も肌トラブルが増える季節。その理由は①花粉による刺激 ②紫外線の急増 ③気温変化によるバリア機能の低下。でも、大丈夫！✨ 適切なケアで予防できます。【春の肌荒れ対策3選】1️⃣保湿を徹底する 2️⃣優しい洗顔を心がける 3️⃣紫外線対策を早めに始める。当サロンでは、春の肌トラブルに特化したフェイシャルコースをご用意しています💆‍♀️」

これらすべて、ライティング能力に基づいています。

【工程5】効果測定と改善（100%言語作業）

投稿後は数字を言語で解釈し、次の改善策を導きます。

【効果測定レポート例：3/5投稿】
リーチ：850人（+30%）
エンゲージメント率：4.2%（目標達成✅）
保存数：42件（過去最高）
分析：実用的な情報は保存されやすい。今後もTips系コンテンツを強化すべき

SNS運用の全工程は、戦略立案・企画・キャプション・分析にいたるまで、すべて言語による設計作業で構成されています。

3-2. 全工程における言語作業の割合

月20投稿のSNS運用の時間配分：

  • 工程1（分析と戦略立案）：180分/月
  • 工程2（カレンダー作成）：60分/月
  • 工程3（個別企画）：20分×20投稿 = 400分/月
  • 工程4（キャプション作成）：15分×20投稿 = 300分/月
  • 工程5（効果測定）：120分/月
  • 画像作成・編集：10分×20投稿 = 200分/月

合計：1,260分（約21時間/月）

このうち、画像作成以外の1,060分（約84%）は、言語を使った企画・執筆・分析です。

第4章：自動化・システム構築の実際のフロー

自動化・システム構築において、最も重要なのは技術ではなく言語設計です。一見技術的に見えるこの分野も、その本質は「どう動くべきか」を言語で設計することにあります。

4-1. メール自動返信システムの構築フロー

「問い合わせメールに自動返信するシステム」を例にとると、作業の大部分が言語設計で占められていることがわかります。

  1. 要件定義（100%言語）：「どんな問い合わせに」「どう返信するか」を設計する。
  2. シナリオ設計（100%言語）：問い合わせのパターンごとに対応を言語化する。
  3. プロンプト設計（100%言語）：AIに「どう判断させるか」「どう返信させるか」を指示する。
  4. テスト設計（100%言語）：どんなケースでテストするかを計画する。
  5. システム構築（50%言語、50%技術）：実装とテストを行う。

自動化システムの本質は、「システムがどう動くべきか」を言語で設計することです。技術的な実装はプロセス全体の一部に過ぎず、言語設計の精度がシステムの品質を左右します。

第5章：まとめ ─ すべての起点は「言語設計」

この章では、画像生成・動画制作・SNS運用・自動化の4つの分野について、実際の作業フローを詳細に分解しました。

5-1. 共通する構造

どの分野でも、以下の構造が共通していることがわかります：

  1. 最初に「何を作るか」を言語で設計する（要件定義・企画）
  2. 詳細な設計図を言語で作る（コンセプト・構成・台本）
  3. AIへの指示を言語で書く（プロンプト・スクリプト）
  4. 出力を言語で評価・判断する
  5. クライアントに言語で説明する

「技術作業」の前後には、必ず「言語設計」が存在する

5-2. 言語設計能力がないとどうなるか

言語設計能力がない状態で、これらの分野に取り組むと：

  • 画像生成：何度やっても意図した画像が出ない
  • 動画制作：構成がバラバラで、メッセージが伝わらない
  • SNS運用：投稿が読まれず、エンゲージメントが上がらない
  • 自動化：システムが期待通りに動かず、修正に時間がかかる

結果として、クライアントからの評価が低く、継続依頼が来ないという状況に陥ります。

5-3. だからこそ、ライティングから始める

この章で見てきたように、すべてのAI副業の「見えない部分」には、言語設計が存在します。そして、この言語設計能力を最も効率的に身につけられるのが、ライティングです。

ライティングで身につく4つの力

  • 「何を伝えるか」を設計する力が身につく
  • 「どう伝えるか」を構造化する力が身につく
  • 「読者の心を動かす」言葉の選択力が身につく
  • 「論理的に説明する」能力が身につく

これらすべてが、画像・動画・SNS・自動化の「言語設計」に直結します。

おわりに

この章では、「画像・動画・自動化・SNSの前に必ず存在する工程」として、言語設計を可視化しました。

重要なポイント

  • どの分野でも、作業時間の80〜95%は言語設計
  • 技術作業は全体のごく一部に過ぎない
  • 言語設計能力がないと、どの分野でも中途半端な結果しか出せない
  • ライティングは、この言語設計能力を身につける最適な方法
