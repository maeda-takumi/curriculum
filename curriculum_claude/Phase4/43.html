第1章：Claudeの「クセ」を理解する【所要時間15分】

Claudeには「クセ」があります。ブログ・記事ライティングで特に気をつけるべきクセを把握しましょう。

1-1. 記事ライティングで出やすいClaudeの「クセ」

【クセ①：導入が長すぎる】「〜についてお悩みではありませんか？〜は非常に重要です。この記事では〜」→ 読者は早く本題に入ってほしい

【クセ②：同じことを言い換えて繰り返す】同じ内容を別の言葉で2〜3回書く → 文字数は増えるが、情報密度が下がる

【クセ③：「いかがでしたか？」で締める】「いかがでしたか？この記事では〜についてご紹介しました」→ 読者に嫌われる定番フレーズ

【クセ④：具体例が抽象的】「例えば、Aさんは〜」と言いながら、具体性がない → 実感が湧かない

【クセ⑤：見出しが説明的すぎる】「〇〇とは何かについての説明」→ 長い、キーワードが埋もれる

【クセ⑥：箇条書きに頼りすぎる】本文のほとんどが箇条書き → 読みにくい、SEO的にもマイナス

【クセ⑦：「思います」「でしょう」が多い】自信なさげな表現が頻出 → 説得力がない

1-2. 【実践】クセ対策をカスタム指示に追加する

【STEP1：記事ライティング用のクセ対策を設定する】

【操作手順】

1. Claudeの設定を開く（左下アイコン → Settings → Custom instructions）

2. 下の欄に以下を追加（既存の内容の後に追記）

【追加する内容：記事ライティングのルール】

  • ・導入は3文以内で本題に入る
  • ・同じ内容の言い換え繰り返し禁止
  • ・「いかがでしたか」「参考になれば幸いです」禁止
  • ・具体例は固有名詞・数字を入れて実感が湧くように
  • ・見出しは簡潔に（15文字以内目安）
  • ・箇条書きは1セクション3〜5項目まで、本文は文章で
  • ・「思います」「でしょう」より「です」「ます」で断定
  • ・冗長な接続詞（「また」「そして」の連続）を避ける

3. 保存

【基本設定】■ 一人称：私 / 僕 / 筆者 / 使わない / ■ 読者への呼びかけ：あなた / 皆さん / ■ 文体：です・ます調 / だ・である調 / 混合

【文の長さとリズム】■ 1文の目安：＿＿文字以内 / ■ 1段落の目安：＿＿文 / ■ リズム：短文でテンポよく / じっくり丁寧に

【語尾のバリエーション】■ 基本：「です」「ます」 / ■ バリエーション：「〜ですよね」使う/使わない / 「〜しましょう」使う/使わない / 「〜なんです」使う/使わない

【禁止事項】・（例：「させていただきます」「いかがでしたか」）・（例：絵文字）・（例：過度なカタカナ語）

第6章：A/Bテストと継続的改善【所要時間15分】

6-1. A/Bテストの考え方

2つのバージョンを比較し、どちらが良い結果を出すか検証する手法。プロンプトでのA/Bテスト例：バージョンA「見出しを5つ作成してください」vs バージョンB「SEOを意識した見出しを5つ、キーワードを含めて作成してください」

テストする価値があるポイント：役割設定の書き方 / 制約条件の厳しさ / 出力形式の指定方法 / 文字数の指定方法 / 具体例を入れるか入れないか
