4-1. ライティングを『使い回す』という発想

フェーズ4｜再現性・自動化・仕組み化編

はじめに：「1回書いたら終わり」から「1回書いたら10回稼ぐ」へ

フェーズ4の幕開けです。フェーズ3であなたは、セールスライティングの型、LP設計、SNS・広告・台本への展開、複数ジャンルへの応用と、「どんな案件でも対応できる」武器を手に入れました。

しかし、ここで1つ残酷な現実をお伝えしなければなりません。

「毎回ゼロから全部作る」やり方では、副業月収30万円が天井になります。

フェーズ3で学んだスキルを使えば、1案件あたりの制作時間は確かに短縮されています。Claudeを駆使すれば、LP＋SNS＋広告の一式を5〜8時間で仕上げられるようになったはずです。

しかし、「毎回ゼロから」では、その5〜8時間は毎回「労働」として消費されます。仮に副業に使える時間が月40時間だとすると、5〜8時間の案件を5〜6件こなすのが限界です。1件8万円の案件を5件で月収40万円。一見十分に見えますが、その40時間は毎月「労働」として拘束され続けます。

フェーズ4のテーマは、この「労働」から少しずつ離れることです。

その第一歩が、この4-1で学ぶ「ライティングを使い回す」という発想です。使い回すといっても、同じ文章を複数のクライアントに売りつけるという話ではありません。一度作ったものから「構造」「表現パターン」「思考プロセス」を抽出し、次の案件で再利用できる状態にするということです。

【この章で手に入るもの】

  • 「毎回ゼロから」を脱却し、過去の仕事を資産化する思考法
  • 構造・表現・情報の3層モデルで「何を使い回せるか」を見極める力
  • 1つの案件から10個のテンプレートを抽出する具体的手法
  • Claudeを使って「使い回し」を加速するプロンプト設計
  • 過去案件を棚卸しして「資産ライブラリ」を構築する実践ワーク
  • 使い回しの倫理と、クライアントへの誠実さを両立させる判断基準

【この章のルール】

  • ① 過去の案件を「消費」ではなく「投資」と捉える
  • ② 「楽をする」ではなく「価値を再生産する」という意識を持つ
  • ③ クライアントごとの機密情報と、抽象化できる構造を明確に分ける
  • ④ 実際に手を動かして「資産ライブラリ」を1つ以上作る

⚠ 注意：この章は「効率化のテクニック」を学ぶ章ですが、根底には「クライアントへの誠実さ」があります。目的は「より少ない労力でより高い価値を提供する」ことであり、「品質を落として楽をする」ことではありません。

第1章：なぜ「使い回す」という発想が必要なのか

1-1. 副業ライターの時間価値の現実

副業ライターとして稼ぐとき、最も重要な指標は「時給」です。月収30万円でも200時間働いていたら時給1,500円。月収15万円でも30時間で稼げていたら時給5,000円。副業の持続可能性という観点では、後者のほうが圧倒的に健全です。

同じスキルレベル、同じ単価でも、「使い回し」ができるかどうかで時給が2倍違います。Bさんは「同じ報酬をより短い時間で稼ぐ」か「同じ時間でより多く稼ぐ」かを選べます。この選択肢の広さこそが最大のメリットです。

1-2. 「毎回ゼロから」が副業を破綻させる理由

【リスク①：品質のばらつきが出る】毎回ゼロから作ると、人間の集中力や体調の波が品質に影響する。型を使えば「最低限の品質」が担保される。

【リスク②：成長が遅くなる】型を使っていれば「この型のここを改善したら反応が良くなった」という学びが蓄積される。

【リスク③：精神的に消耗する】毎回構成・流れ・表現パターンをゼロから決めると「決断疲れ」が起きる。型を使えば本当に重要な判断にエネルギーを集中できる。

【リスク④：スケールできない】型やテンプレートがあれば「この型で作って」と外注・チーム化への道が開ける。

【リスク⑤：競争力を失う】AI時代において「ゼロから作る力」の価値は相対的に下がっている。Claudeを使えば誰でもそこそこの文章は書けるからです。差別化できるのは「自分だけの型」「自分だけのテンプレート」「過去の成功パターンの蓄積」です。これらは「使い回し」を前提とした仕事の仕方でしか手に入りません。

「毎回ゼロから」は、一見すると「誠実な仕事ぶり」に見えますが、実は「持続可能性」「品質安定」「成長速度」のすべてを犠牲にしている非効率な働き方なのです。

1-3. 優秀なビジネスパーソンはどうしているか

優秀なコンサルタントは過去の資料を「テンプレート化」し、新しいプロジェクトでゼロから資料を作ることはありません。優秀な営業担当者は「売れるトークスクリプト」を持っています。優秀なプログラマーは「スニペット」や「ライブラリ」を自分で作り、再利用しています。

ライティングも同じです。優秀なコピーライターは、過去に書いた文章から「型」「表現」「構造」を抽出し、次の案件で再利用しています。これは「手抜き」ではなく「プロの仕事術」です。

💡 「使い回し」のプロの思考法：「この案件は終わった」ではなく「この案件から何を抽出できるか」と考える。「納品したら終わり」ではなく「納品＋テンプレート化」までが1セット。このサイクルが身につくと、案件をこなすほど「資産」が増えていきます。

1-4. ライティングを「資産」と捉える発想転換

ライティングの仕事は、「納品物を作る」ことではなく「資産を生み出す」ことです。

1つの案件で作成したLP原稿には以下の「資産」が含まれています：

【資産①：構造テンプレート】LPの「構成」は別の案件でも使えます。

【資産②：表現パターン】うまくいった見出し、フック、CTA文言などは別の案件でも応用できます。

【資産③：思考プロセス】「このターゲットにはこういうアプローチが刺さった」という知見は類似ターゲットの案件で再利用できます。

【資産④：Claudeプロンプト】その案件でうまくいったプロンプトは別の案件でも使えます。「この業種のLPはこのプロンプトで生成する」という知見が蓄積されます。

【資産⑤：業界知識】その案件で得た業界特有の知識（法規制、専門用語、ターゲットの悩み）は同業種の次の案件で再利用できます。

1つの案件を「納品して終わり」にするか、「5つの資産を抽出して次に活かす」かで、1年後の生産性は10倍変わります。

2-2. 案件をまたいで使い回せる「構造」とは

第1層の「構造」は最も使い回しやすい要素です。「人間の心理」に基づいているため業種を超えて使えます。

【使い回せる構造の例】

  • LP構成：問題提起 → 共感 → 解決策 → 証拠 → 行動喚起
  • YouTube台本：フック → 橋渡し → 本編 → まとめ → CTA
  • SNS投稿：結論ファースト → 理由 → 具体例 → CTA
  • 広告文：フック → ベネフィット → CTA
  • メルマガ：近況 → 価値提供 → オファー → CTA

【LP構成テンプレート（汎用版）】

1. ファーストビュー：キャッチコピー ＋ サブコピー ＋ CTA

2. 問題提起：ターゲットの悩み・不安を言語化

3. 共感：「その気持ち、わかります」パート

4. 解決策提示：サービス/商品の紹介

5. 特徴・メリット：3〜5つの差別化ポイント

6. 証拠：実績・数字・第三者評価

7. お客様の声：3名分

8. 比較：競合との違いを明示

9. よくある質問：5問

10. 最終CTA：「今すぐ〜」の行動喚起

この構造は、どんな業種でもそのまま使えます。変わるのは「各セクションに入れる情報」だけです。

2-3. 業種をまたいで使い回せる「表現パターン」

【フック（注目を引く）表現パターン】

  • 「〜していませんか？」（問いかけ型）
  • 「たった〇〇で△△が変わる」（変化型）
  • 「〇〇の方へ」（ターゲット明示型）
  • 「知らないと損する〇〇」（損失回避型）
  • 「〇〇が選ばれる3つの理由」（根拠型）

【CTA（行動喚起）表現パターン】

  • 「今すぐ〜」（即時性）/ 「無料で〜」（リスク排除）
  • 「限定〇〇名」（希少性）/ 「〇日間返金保証」（安心感）

【証拠（信頼性）表現パターン】

  • 「〇〇%の方が満足」（数字型）/ 「〇〇で紹介されました」（メディア型）
  • 「〇〇専門家も推薦」（権威型）/ 「累計〇〇名が体験」（実績型）

⚠ 表現の使い回しで注意すべき点：業種特有の言い回し・専門用語は個別に調整が必要。法規制がある業種（美容・健康・金融）は表現に制限があります。同じパターンを多用しすぎると「どこかで見た文章」になるため、パターンを「組み合わせる」ことでオリジナリティを出しましょう。

2-4. 絶対に使い回してはいけないもの

【絶対に使い回してはいけない「情報」】

  • クライアント固有の社名・商品名・サービス名
  • 実績数字（「導入企業2,500社」など）
  • お客様の声（実在の顧客からの声）
  • 具体的な価格・キャンペーン情報
  • 競合比較で使った他社の情報
  • クライアントから受領した機密情報
  • 特許・商標に関わる表現

特に注意すべきは「お客様の声」です。A社の案件で作成したお客様の声を、B社の案件で流用することは絶対にNGです。

「構造」と「表現パターン」は使い回し、「情報」は毎回新しく作る。この原則を徹底してください。

第3章：1つの案件から「10倍」引き出す実践テクニック

3-1. 案件完了後の「資産化」5ステップ

案件を納品したら、すぐに次の作業に取り掛かるのではなく、15〜20分だけ「資産化」の時間を取ってください。この習慣が、半年後・1年後のあなたの生産性を劇的に変えます。

【STEP1：構造を抽出する（5分）】納品した成果物の「構造」を言語化します。LPなら「何セクションで、各セクションの目的は何だったか」。この構造を「テンプレート」としてドキュメントに残します。

【STEP2：表現パターンを抽出する（5分）】うまくいった表現をピックアップします。キャッチコピー、見出し、CTA文言、フックを抜き出し、「なぜこの表現がうまくいったか」の理由も一言メモしておきます。

【STEP3：Claudeプロンプトを保存する（3分）】その案件で使ったプロンプトの中で「これは使える」と思ったものを保存します。「最初に投げるプロンプト」「修正を依頼するときのプロンプト」「最終確認のプロンプト」の3種類は必ず残しましょう。

【STEP4：業界知識をメモする（3分）】その案件で得た「業界固有の知識」をメモします。法規制、専門用語、ターゲットの特性、競合の傾向など。

【STEP5：振り返りを1行書く（2分）】「この案件でうまくいったこと・改善点」を1行だけ書きます。「共感パートを厚めにしたらクライアント評価が高かった」など、次に活かせる一言で十分です。

第4章：Claudeで「使い回し」を加速する

ここまでは「人間の頭」で使い回しを考える方法でした。ここからは、Claudeを使って「使い回し」を加速させるテクニックを紹介します。Claudeは、「抽出」「変換」「蓄積」のすべてで活躍します。

4-1. 構造抽出プロンプトの使い方

過去に作成した原稿から「構造」を抽出するとき、Claudeを使うと効率的です。以下のプロンプトを使ってみてください。

【構造抽出プロンプト（コピペ用）】

以下のLP原稿を分析し、「汎用的に使える構造テンプレート」を抽出してください。【LP原稿】（ここに原稿を貼り付け）

【出力形式】1.全体構成：何セクションで構成されているか、各セクションの目的を記載 / 2.心理フロー：読者の心理がどう変化していくか / 3.各セクションのテンプレート：業種を特定しない汎用的な形で記載 / 4.使い回しポイント：この構造を別の業種で使うときに変更すべき点と、そのまま使える点を明記

クライアント固有の情報（社名、商品名、数字）は含めず、「〇〇」「△△」などのプレースホルダーで表現してください。

このプロンプトを使うと、Claudeが原稿を分析し、「業種を入れ替えても使える構造テンプレート」を生成してくれます。出力されたテンプレートを保存しておけば、次の案件でそのまま使えます。

4-2. 業種変換プロンプトの設計

抽出した構造テンプレートを、別の業種に適用するときに使う「業種変換プロンプト」です。

【業種変換プロンプト（コピペ用）】

以下の「構造テンプレート」を使って、【業種名】業界のLP原稿を作成してください。

【構造テンプレート】（構造抽出プロンプトで生成したテンプレートを貼り付け）

【業種情報】業種：/商品/サービス名：/価格：/ターゲット：/特徴・差別化ポイント：

【条件】・テンプレートの構造・流れを忠実に再現 ・業種固有の専門用語・表現を適切に使用 ・ターゲットの悩み・欲求を業種に合わせて具体化 ・法規制がある場合は表現に注意

4-3. 「成功した表現」をClaudeに学習させる方法

過去の案件で「うまくいった表現」をClaudeに覚えさせ、次の案件で活用する方法です。

Claudeには「長期記憶」はありませんが、プロンプト内に「参照情報」として渡すことで同様の効果を得られます。

【成功表現活用プロンプト（コピペ用）】

あなたは優秀なコピーライターです。以下の「成功した表現集」を参考にして、【今回の案件】のコピーを作成してください。

【成功した表現集】＜キャッチコピー＞・「〇〇な毎日を、△△に変える」→ 教育スクールで反応率2.3% ・「たった□□で、●●が変わった」→ 美容商品でCVR1.8% （過去にうまくいった表現と、その成果を列挙）

【今回の案件情報】（案件情報を記載）

【条件】・成功した表現のパターンを参考にしつつ、今回の案件に最適化 ・丸コピーではなく、エッセンスを活かした新しい表現に ・なぜその表現パターンを採用したかの理由も簡潔に説明

「成功した表現集」をNotionやGoogleドキュメントに蓄積しておき、案件のたびにこのプロンプトに貼り付けて使います。成功パターンが増えるほど、Claudeの出力品質が上がります。

4-4. 自分専用の「表現データベース」を作る

「使い回し」を本格的に実践するには、自分専用の「表現データベース」を構築することが不可欠です。

【表現データベースの構造】

  • カテゴリ別に整理：キャッチコピー（フック系 / 疑問系 / 数字系 / 共感系）/ 見出し（問題提起 / 解決策 / 証拠 / CTA）/ CTA文言（即時性 / 希少性 / リスク排除 / ハードル低減）
  • 各表現に付与する情報：表現の原文 / 使った業種/案件 / 成果（CVR、反応率など）/ 使える場面のタグ（LP / SNS / 広告 / メール）/ 改善メモ
  • 保存場所：Notion（検索・フィルタが便利）/ Googleスプレッドシート（一覧性が高い）/ Obsidian（タグでの関連付けが強力）

まずは「案件が終わるたびに3つの表現を追加する」というルールで始めてください。3ヶ月で30〜50個の表現が蓄積されれば、十分に「使い回し」が機能し始めます。

第6章：使い回しの倫理と注意点

6-1. クライアントに対する誠実さ

【OK】構造やパターンを活用し、クライアント固有の情報で「カスタマイズ」する → プロとして当然の効率化。

【OK】過去の成功事例を参考に、「より良い提案」をする → 経験を活かすのはプロの仕事。

【NG】他のクライアントの成果物を、そのままコピーして納品する → これは詐欺。絶対にやってはいけない。

【NG】「ゼロから作りました」と嘘をつく → テンプレートを使っていることを隠す必要はない。

【グレー】同じ業種の競合クライアントに、類似した構造を使う → 「差別化ポイント」が被らないよう配慮する。

6-2. 守秘義務と著作権の境界線

【守秘義務（NDA）の観点】クライアントから受領した情報は絶対に他のクライアントに渡さない / 「構造」を抽出するときも、固有情報が残らないよう注意

【著作権の観点】あなたが書いた文章の著作権は契約次第でクライアントに帰属する場合がある / ただし「構造」や「アイデア」は著作権の対象外（表現のみが対象）/ 「構造テンプレート」は著作権的には問題なく使い回せる

【契約書の確認ポイント】著作権の帰属条項 / 秘密保持条項 / 競業避止条項 / 不明な場合はクライアントに確認するのが最も安全

6-3. 「手抜き」と「効率化」の違い

手抜き（NG）

効率化（OK）

過去の成果物をそのままコピペして納品

過去の「構造」を活用し、新しい「情報」で作り直す

クライアントの特性を無視して汎用的な文章を渡す

テンプレートをクライアントに合わせてカスタマイズ

時間を短縮した分、品質が下がる

時間を短縮した分、より深い分析に時間を使う

「バレなければいい」という意識

「プロの技術として胸を張れる」という意識
